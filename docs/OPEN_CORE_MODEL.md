# Open-Core Model

Nexus is public where users and contributors benefit from transparency: clients, shared client runtime, user-facing documentation, tests and release tooling.

Nexus is private where user accounts, monetization, cloud data and operational safety require server-side control.

## Public Core

- Local-first workspace features
- Client UI and shell behavior
- Shared runtime contracts
- Client-side safety guardrails
- Public release packaging
- Public documentation and contribution flow

## Private Cloud

- Account and auth implementation
- Billing and payment processing
- Sync, backup and cloud storage
- AI/Flux orchestration
- Team and sharing controls
- Pro entitlement enforcement
- Admin/control operations
- Infrastructure, abuse controls and secrets

## Rule

The public client can present feature availability, but Nexus Cloud is the authority for cloud and Pro access.
